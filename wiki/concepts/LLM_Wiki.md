# LLM Wiki

## 概述

LLM Wiki 是前 OpenAI 科学家 Andrej Karpathy 推出的个人知识库构建方案，于 2026 年 4 月发布，引发社区热议。

## 核心理念

在 Agent 时代，只需分享"**想法文件**"，由 Claude、Codex 等 Agent 自动构建个人知识库。

## 系统架构

### 三层结构

```
┌─────────────────┐
│   Schema 层     │  ← 知识结构和关系定义
├─────────────────┤
│   Wiki 层       │  ← AI 生成的知识库页面
├─────────────────┤
│   原始数据层    │  ← 文档、笔记、资料
└─────────────────┘
```

### 数据流

```
原始资料 → 数据摄取 → Wiki 生成 → 质量检查 → Schema 验证
     ↑                                          ↓
     └──────────── 自我增强循环 ────────────────┘
```

## 核心流程

### 1. 数据摄取
- 文档导入
- 笔记整理
- 资料分类
- 元数据提取

### 2. 查询处理
- 自然语言查询
- 语义检索
- 跨文档关联
- 答案生成

### 3. 质量检查
- 事实核查
- 链接验证
- 时效性检查
- 冲突检测

## 应用场景

| 场景 | 描述 |
|------|------|
| 研究 | 学术论文和资料管理 |
| 阅读 | 读书笔记和知识整理 |
| 企业知识管理 | 组织内部知识积累 |
| 个人知识库 | 个人知识体系构建 |

## 技术特点

### 持续积累
- 知识不断累积
- 自我增强
- 版本控制
- 更新追踪

### Agent 协作
- Claude 自动构建

### 开放生态
- 开源方案

## 目录结构示例

```
llm-wiki/
├── raw/           # 原始资料
│   ├── articles/  # 文章
│   ├── papers/    # 论文
│   └── assets/    # 附件
├── wiki/          # AI 生成的 Wiki
│   ├── concepts/  # 概念
│   ├── entities/  # 实体
│   ├── sources/   # 摘要
│   └── comparisons/ # 对比
└── index.md       # 索引
```

- **知识民主化**：让每个人都能构建个人知识库
- **Agent 时代基础设施**：为 AI Agent 提供知识基础
- **开源社区驱动**：社区共同建设和改进
