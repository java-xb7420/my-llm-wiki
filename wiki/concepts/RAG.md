# RAG（检索增强生成）

## 定义

**Retrieval-Augmented Generation（RAG）** 是一种将检索机制与大语言模型结合的技术范式。在回答用户问题时，模型先从外部知识库中检索相关文档片段，再基于检索结果生成答案。

## 工作原理

```
用户查询 → 向量检索/关键词检索 → 相关文档片段 → LLM 生成答案
```

1. **文档分块**：将知识库文档切分为较小的片段（chunk）
2. **向量化**：使用嵌入模型将片段转换为向量，存入向量数据库
3. **检索**：用户查询时，将查询向量化并检索最相似的片段
4. **生成**：将检索到的片段作为上下文，与查询一起送入 LLM 生成答案

## 优势与局限

**优势**：
- 无需微调模型即可利用外部知识
- 答案可追溯到原始来源
- 知识库可随时更新

**局限**（在个人知识库场景下）：
- 每次查询都从零开始检索，无知识积累
- 复杂问题需跨多文档综合时，检索效果受限
- 模型不维护文档间的关联关系，每次重新拼接碎片
- 答案质量受检索精度影响显著

## 与 LLM Wiki 的对比

RAG 是**查询时检索**——每次回答都重新发现知识。[[LLM Wiki]] 是**持久化构建**——LLM 维护一份不断更新的结构化 Wiki，知识编译一次并持续累积。RAG 适合一次性问答；LLM Wiki 适合长期知识积累。
