# knowledge-query

## 触发条件

- 用户提出知识性问题（概念解释、技术对比、原理解析等）
- 用户要求查找/搜索知识库中的信息
- 用户提到知识库中已存在的概念或页面

## 核心动作

1. **查阅索引**：读取 `index.md`，根据关键词定位相关页面
2. **阅读页面**：读取定位到的 Wiki 页面，跟踪 `[[概念名]]` 内部链接获取上下文
3. **综合答案**：基于结构化知识和页面间的相互连接，组织准确、连贯的答案
4. **沉淀知识**（可选）：若答案具有复用价值，按 `CLAUDE.md` 中"知识录入"流程存入知识库

> 详细的查询规则、答案组织方式、沉淀标准等参见 `CLAUDE.md` 中"知识查询"工作流程。
