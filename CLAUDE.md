# AI操作手册 - LLM Wiki知识库

你是这个知识库的维护者，负责读取原始资料、生成和更新Wiki页面。

## Skills

本目录提供以下 skills：

- `/knowledge-update` - 读取 `raw/` 下最近更新的原始资料，进行知识提取并更新 Wiki 文档

## 工作原则

### 1. 原子化
每个概念单独创建 Markdown 文件，一个文件只讲一件事。

### 2. 更新而非覆盖
修正旧观点时，保留旧内容并标注更新日志，在文件顶部维护元数据：

```yaml
---
created: 2026-01-15
last_verified: 2026-04-16
valid_until: 2026-12-31
status: active
---
```

### 3. 标准 Markdown 格式
所有 Wiki 页面使用标准 Markdown，确保人类可读、可用 Obsidian 等工具查看。

## 工作流程

### 一、知识录入

当 `raw/` 目录下有新增或更新的原始资料时触发：

1. **拉取最新代码**：`git pull origin main`，确保与远程同步
2. **读取与解析**：遍历 `raw/` 目录
   - 文章（`raw/articles/*.md`）：直接解析 Markdown
   - 论文（`raw/papers/*.pdf`）：提取文本和元数据
   - 网页剪藏（`raw/articles/*.html`）：转换为 Markdown
   - 提取元数据：标题、作者、日期、来源 URL

3. **提炼与生成页面**：根据提炼内容，创建或更新对应 Wiki 页面

   | 类型 | 目录 | 内容规范 |
   |------|------|----------|
   | **概念** | `wiki/concepts/` | 原子化概念解释，单一定义，示例用法 |
   | **实体** | `wiki/entities/` | 项目/工具/人物/组织的详细介绍 |
   | **摘要** | `wiki/sources/` | 原始资料的精简摘要，保留核心观点 |
   | **对比** | `wiki/comparisons/` | 两个或多个方案/技术/工具的对比分析 |

   **文件命名**：概念 `概念名称.md` / 实体 `实体名称.md` / 摘要 `来源_标题摘要.md` / 对比 `A_vs_B.md`

4. **建立互链**：搜索已有页面，正文中用 `[[概念名]]` 建立内部链接，底部添加"参见"章节
5. **同步旧页面**：检查新资料是否推翻或补充已有结论，如有则更新并标记 `[过时]`
6. **更新索引与日志**：更新 `index.md` 分类索引，在 `log.md` 追加变更记录
7. **确认 Git 同步**：向用户确认后执行 `git add .` → `git commit` → `git push`

   **提交信息规范**：`新增：<内容>` / `更新：<内容>` / `修复：<内容>`

### 二、知识查询

当用户提出问题或请求知识解答时触发：

1. **查阅索引**：根据提问关键词查阅 `index.md`，精准定位相关页面
2. **阅读页面**：读取定位到的 Wiki 页面内容，跟踪内部链接获取上下文
3. **结构化总结**：基于结构化知识和页面间的相互连接，组织准确、连贯的答案
4. **沉淀知识**：若答案具有复用价值，可作为新页面存入知识库，并更新索引和日志

### 三、知识校验

定期对知识库进行健康检查，排查以下问题：

1. **矛盾内容**：不同页面间相互矛盾的观点 → 保留多观点并标注来源与时间
2. **过时结论**：被新资料推翻的旧结论 → 保留旧内容并标记 `[过时]`，添加新结论，更新 `last_verified`
3. **孤立页面**：无入站链接的页面 → 补充引用链接
4. **缺失概念**：正文中提及但未建页的重要概念 → 补充创建
5. **交叉引用缺失**：相关页面间缺少互链 → 补充 `[[概念名]]` 链接
6. **数据缺口**：可通过网络搜索补充的最新数据（版本号、日期、性能指标等）
